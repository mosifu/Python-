## 2. tuple 元组：不可变序列的妙用

### 2.1 核心机制

元组的本质是 **"写死的列表"** —— 一旦创建，内容不可增删改。这决定了它只有两个方法（`.count()` 和 `.index()`），没有 `append`/`pop`/`sort` 等。

```python
t1 = (1, 234, 2, 43, 23, '456', 2)
t1.count(2)   # 2
t1.index(2)   # 2
```

### 2.2 组包与解包（元组最常用的场景）

这是元组区别于 list 的最大亮点：

- **组包**：多个值自动打包为元组（括号可省略） `t = 12, 213, 321`
- **基础解包**：`a, b, c = t` —— 变量数量必须与元素数量一致
- **扩展解包**：`first, second, *others = t` —— `*` 收集剩余元素为 **list**
- **经典应用**：`a, b = b, a` 一行交换变量（背后是隐式组包 + 解包）

### 2.3 单元素元组的陷阱

```python
(100)    # int
(100,)   # tuple  ← 逗号是元组的灵魂
```

### 2.4 实战：学生成绩单

用嵌套元组存储多条学生记录，通过解包遍历实现各科统计，是元组的典型应用场景。

```python
for sid, name, chinese, math, english in students:
    total = chinese + math + english
    avg = total / 3
```

## 3. set 集合：去重与集合运算利器

### 3.1 核心特征

- **自动去重**：`{1, 2, 3, 4, 5, 2, 4}` → `{1, 2, 3, 4, 5}`
- **无序**：不能索引，不能切片
- **可变**：`add`/`remove`/`pop`/`clear`

### 3.2 关键坑：空集合 vs 空字典

```python
{}       # 空字典
set()    # 空集合
```

### 3.3 集合运算（set 的核心优势）

用数学运算符替代方法调用，比 Java 更直观：

| 运算 | 运算符 | 方法版 | 含义 |
|------|--------|--------|------|
| 交集 | `s1 & s2` | `s1.intersection(s2)` | 两者都有 |
| 并集 | `s1 \| s2` | `s1.union(s2)` | 合并去重 |
| 差集 | `s1 - s2` | `s1.difference(s2)` | 在 s1 不在 s2 |

### 3.4 集合推导式

```python
# 语法：{表达式 for 元素 in 集合 if 条件}
only_football = {s for s in football if s not in basketball}
```

和列表推导式 `[x for x in ...]` 类似，只是方括号换花括号，结果自动去重。

### 3.5 选课统计案例

通过四组学生名单（足球/篮球/法语/美术）进行集合运算，实现了：
- 交集找多门全选的学生
- 差集找只选某课程的学生
- 并集得到所有学生列表
- 利用 `list.count()` 统计每人选课数

## 4. dict 字典：键值映射结构

### 4.1 核心特征

- **键值对**：每个 key 对应一个 value
- **key 不可重复**：重复赋值覆盖旧值
- **key 必须可哈希**：str/int/float/tuple 可以，list/set/dict 不行
- **3.7+ 插入有序**
- **O(1) 查找**（哈希表实现）

### 4.2 增删改查

| 操作 | 写法 | 说明 |
|------|------|------|
| 增/改 | `d["key"] = value` | key 不存在则添加，存在则修改 |
| 查 | `d["key"]` | key 不存在报 `KeyError` |
| 安全查 | `d.get("key", default)` | 不存在返回 None 或默认值 |
| 删（返回） | `d.pop("key")` | 删除并返回 value |
| 删（不返回） | `del d["key"]` | 只删除 |

### 4.3 遍历方式

```python
# 方式1：遍历 key
for k in d.keys():
    print(f"{k}: {d[k]}")

# 方式2：遍历 key+value（推荐）
for k, v in d.items():
    print(f"{k}: {v}")
```

### 4.4 嵌套字典（重点）

嵌套字典是实际业务中最常用的数据结构之一，典型结构：

```python
shop = {
    "苹果": {"price": 5.5, "num": 10},
    "香蕉": {"price": 3.0, "num": 20},
}
```

访问嵌套值时注意引号冲突：

```python
# 外层双引号，内层单引号
print(f"单价{info['price']}元, 数量{info['num']}个")
```

## 5. 常见坑点（下半部分合集）

| # | 坑点 | 错误写法 | 正确写法 |
|---|------|---------|---------|
| 1 | 单元素元组 | `(100)` | `(100,)` |
| 2 | 空集合 | `{}` | `set()` |
| 3 | dict key 用可变类型 | `d[[1,2]]` | `d[(1,2)]`（用 tuple） |
| 4 | f-string 嵌套引号 | `f"...{d["key"]}..."` | `f"...{d['key']}..."` |
| 5 | set.remove 不存在元素 | `s.remove(x)`（报错） | `s.discard(x)`（不报错） |
| 6 | dict 安全取值 | `d["key"]`（可能 KeyError） | `d.get("key")` |

## 7. 综合案例深度分析

### 7.1 购物车系统

## 8. 五大容器总对比

| 维度 | list | tuple | set | dict | str |
|------|:----:|:-----:|:---:|:----:|:---:|
| 符号 | `[]` | `()` | `{}`/`set()` | `{k:v}` | `""`/`''` |
| 有序性 | 是 | 是 | 否 | 是(3.7+) | 是 |
| 可变性 | 可变 | **不可变** | 可变 | 可变 | **不可变** |
| 可重复 | 可重复 | 可重复 | **不可重复** | key 不重复 | 可重复 |
| 索引 | 数字索引 | 数字索引 | 不支持 | key 索引 | 数字索引 |
| 适用场景 | 有序集合 | 不变数据/解包 | 去重/交并差 | 键值映射 | 文本处理 |

## 9. 今日总结与下一步

### 学习收获

1. **tuple** 的不可变特性适合存储不应修改的数据，组包/解包是 Python 的核心语法糖，`*` 扩展解包为变量赋值提供了极大的灵活性

3. **dict** 的嵌套结构是构建业务模型的基础，`.get()` 安全取值应该成为习惯

### 遗留问题关注

- `set` 的 `pop()` 是随机删除，实际业务中慎用
- `dict` 的 `.keys()`/`.values()`/`.items()` 返回的是视图对象（view），不是列表，但可迭代
- 嵌套 dict 的引号冲突在复杂 f-string 中容易出错，可考虑用 `.format()` 或模板字符串替代
