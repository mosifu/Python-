## 二、list 列表 — 详细分析

### 2.1 基础操作

列表作为 Python 中最核心的容器类型，本次学习涵盖了完整的增删改查体系：

- **定义与索引**：支持混合类型存储（`[1, 3, 4.5, 'hello', True]`），正向索引从 `0` 开始，反向索引从 `-1` 开始
- **切片**：`list[start:end:step]` 三要素，end 不包含，支持负步长反转（`[::-1]`）
- **增删改查方法链**：
  - 追加 `append()` / 插入 `insert()`
  - 按值删除 `remove()` / 按索引删除 `pop()` 和 `del`
  - 排序 `sort()` / 反转 `reverse()` / 查索引 `index()` / 计数 `count()`



### 2.2 合并与去重

列表合并有三种方式：


| 方式       | 语法              | 特点          |
| -------- | --------------- | ----------- |
| `+` 拼接   | `[1,2] + [3,4]` | 直观，生成新列表    |
| 解包       | `[*a, *b]`      | Pythonic，灵活 |
| `extend` | `a.extend(b)`   | 原地修改        |


去重采用经典「遍历 + `not in`」法，逻辑清晰且保留顺序。

### 2.3 列表推导式

这是本次学习的重要亮点。列表推导式将 `for` 循环和条件筛选压缩为一行表达式：

```
[表达式 for i in 序列 if 条件]
```

典型应用：

- `[i**2 for i in range(1, 21)]` — 生成平方数列
- `[i for i in range(1, 31) if i % 2 == 0]` — 筛选偶数
- `[i**2 for i in range(1, 31) if i % 3 == 0 or i % 5 == 0]` — 复合条件筛选

> 相比 Java 的 Stream API（`map().filter().collect()`），Python 的列表推导式语法更紧凑。

## 三、str 字符串操作 — 详细分析



### 3.1 不可变性与索引

字符串是 **不可变** 序列 —— 任何修改操作都返回新字符串，原字符串不变。这与 Java 的 `String` 类完全一致。

```python
s = "Hello-Python"
# s[0] = "h"     # TypeError!
s = s.replace("H", "h")  # 正确方式
```



### 3.2 切片深入

字符串切片与 list 完全一致，重点学习了**负步长切片的陷阱**：

- `str[0:5:-1]` 返回空字符串（正向 start 方向向后，但 step 为负要求向前，方向冲突）
- `str[-8:-13:-1]` 正确：从 -8 开始向左走到 -13



### 3.3 10 种常用方法


| 方法             | 作用     | 参数       | 返回值           |
| -------------- | ------ | -------- | ------------- |
| `find()`       | 查找子串索引 | 子串       | int（未找到返回 -1） |
| `count()`      | 统计出现次数 | 子串       | int           |
| `split()`      | 切割字符串  | 分隔符      | list          |
| `upper()`      | 全转大写   | 无        | 新 str         |
| `lower()`      | 全转小写   | 无        | 新 str         |
| `strip()`      | 去两端空白  | 无        | 新 str         |
| `replace()`    | 替换子串   | old, new | 新 str         |
| `startswith()` | 检查前缀   | 子串       | bool          |
| `endswith()`   | 检查后缀   | 子串       | bool          |




### 3.4 实战案例

**邮箱验证**：`email.count("@") == 1 and "." in email` —— 简洁而有效。

**回文判断**：利用切片对称性，验证 `text[:half] == text[-1:-half-1:-1]`，比 Java 的双指针检查更 Pythonic。

## 五、与 Java 对比总结

本次学习大量使用 Java 类比，有助于深入理解 Python 的设计哲学：


| 对比维度   | Python                 | Java                                  |
| ------ | ---------------------- | ------------------------------------- |
| 列表定义   | `[1, 2, 3]`            | `new ArrayList<>(List.of(1,2,3))`     |
| 类型约束   | 可混合类型                  | 泛型约束                                  |
| 索引语法   | `s[0]`                 | `s.get(0)`                            |
| 排序     | `list.sort()` 原地       | `Collections.sort(list)` 原地           |
| 筛选     | 列表推导式                  | Stream API `map().filter().collect()` |
| 字符串不可变 | 完全一致                   | 完全一致                                  |
| 字符串方法  | `find()` / `replace()` | `indexOf()` / `replace()`             |


**核心感悟**：Python 在**语法简洁**上胜过 Java，尤其是在数据容器操作方面 —— 列表推导式一行代码即可完成 Java 需要多行的筛选映射操作。

## 六、常见坑点总结

1. `.sort()` **返回** `None` — 原地排序，不要写成 `new_list = list.sort()`
2. **列表越界** — 修改索引超出范围报 `IndexError`，切片越界只返回空列表
3. **字符串不可修改** — `s[0] = 'x'` 报 `TypeError`，应使用 `replace()` 或切片拼接
4. **负步长方向陷阱** — `str[0:5:-1]` 返回空，start 必须在 end 右边
5. **整除运算符** — 回文判断中 `len(text) // 2` 是整除，不能用 `/`
6. **推导式可读性** — 超过 2 层嵌套或复杂条件时，应拆分为普通 `for` 循环
