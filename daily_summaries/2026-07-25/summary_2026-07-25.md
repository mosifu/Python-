## 二、函数基础（第57-63集）

### 2.1 函数定义与调用

Python 函数使用 `def` 关键字定义，后接函数名、括号和冒号，函数体缩进。这与 Java 使用 `public static` + 花括号形成鲜明对比。记号上更简洁，语法上更灵活——Python 函数是**一等公民**，可以赋值给变量、作为参数传递、独立于类存在。

```python
def greet():
    print("hello world")
```

关键规则：**必须先定义后调用**，解释型语言自上而下执行，不同于 Java 编译时的顺序无关性。

### 2.2 参数传递

支持单参数和多参数，参数通过逗号分隔。示例中 `circle_area(r)` 和 `rectangle_area(length, width)` 展示了最基本的用法。

### 2.3 多值返回 —— Python 的独特优势

这是今天印象最深的对比点之一。Python 可以通过 `return a, b` 直接返回多个值，底层自动打包为元组。调用方可以选择：
- 用一个变量接收（得到一个元组）
- 用解包接收（`area, perimeter = circle_info(5)`），更加直观

Java 要实现类似效果必须手动创建包装类或使用集合，Python 一行搞定。

### 2.4 函数嵌套调用

函数内调用另一个函数，执行顺序呈现**栈式结构（LIFO）**——调用时依次入栈，返回时依次出栈。示例中 `a() -> b() -> c()` 的输出顺序：

```
a...before -> b...before -> c... -> b...after -> a...after
```

这为后续理解递归调用奠定了直觉基础。

### 2.5 Docstring

使用三引号在函数体开头编写文档字符串，遵循 `:param` 和 `:return` 约定。虽然是 Python 的"软规范"而非强约束，但写 docstring 是专业开发者的基本素养。

## 三、传参方式（第64-66集）

Python 提供三种传参方式，灵活性远超 Java：

| 方式 | 语法 | 说明 |
|------|------|------|
| **位置传参** | `func(1, 2, 3)` | 按函数定义的参数顺序一一对应 |
| **关键字传参** | `func(a=1, b=2)` | 指定参数名传参，顺序可任意 |
| **混合传参** | `func(1, 2, c=3)` | 位置参数必须在关键字参数之前 |

**硬性规则**：*位置参数必须在关键字参数之前*，违反即 SyntaxError。

### 默认参数

默认参数在函数定义时赋值，调用时可省略。默认参数必须放在非默认参数的后面：

```python
def register(name, age, gender="男", city="东莞")
```

这让一个函数等效于 Java 中多个重载方法，极大减少了代码量。

## 四、变量作用域（第64集）

| 作用域 | 关键字 | 说明 |
|--------|--------|------|
| **局部变量** | --- | 函数内部定义，函数外不可访问 |
| **全局变量** | `global` | 在函数顶部声明后，可读可写全局变量 |

如果不加 `global`，函数内对同名变量赋值会创建新的局部变量，全局变量不受影响。反之，使用 `global num` 声明后，函数内的修改会直接影响全局变量。

> **PEP 8** 建议 `global` 声明放在函数体最前面。

## 五、不定长参数（第66集）

这是 Python 函数灵活性的集中体现：

### `*args` --- 可变位置参数 -> 元组

```python
def calc_data(*args):
    return min(args), max(args), round(sum(args) / len(args), 2)
```

接收任意个数的位置参数，打包成元组。这在不确定参数个数时极其有用。

**Java 对比**：`void func(int... args)` 可变参数，功能类似但 Python 更简洁。Java 只能用于同类型参数，Python 的 `*args` 能接收任意类型。

### `**kwargs` --- 可变关键字参数 -> 字典

```python
def calc_data2(*args, **kwargs):
    if kwargs.get("round") is not None:
        avg_val = round(avg_val, kwargs["round"])
    if kwargs.get("print"):
        print(...)
```

接收任意个数的关键字参数，打包成字典。典型用途是传递可选配置参数。

**Java 对比**：无直接语法等价物，通常使用 `Map<String, Object>` 或 Builder 模式变通实现。

## 六、函数进阶特性（第68-70集）

### 6.1 函数作为参数（高阶函数）

Python 函数是一等公民，可以像普通变量一样传递。核心模式：

```python
def calc(x, y, operation):
    return operation(x, y)

calc(2, 3, add)       # 5
calc(2, 3, subtract)  # -1
```

**Java 对比**：Java 8+ 通过函数式接口 + Lambda 实现类似效果（如 `BiFunction<Integer, Integer, Integer>`），但语法冗余度更高。

### 6.2 lambda 匿名函数

| 维度 | `def` 命名函数 | `lambda` 匿名函数 |
|------|:--:|:--:|
| 代码量 | 多行 | 单行表达式 |
| 可复用 | 是 | 否，用完即弃 |
| 典型场景 | 通用逻辑封装 | sort 的 key、filter 的条件 |
| 副作用 | 支持 print 等 | 不推荐有副作用 |

核心用法：`lambda 参数: 返回值表达式`

最经典的应用场景是 `sort(key=)`：
```python
data.sort(key=lambda item: len(item))
```

### 6.3 递归函数

以**阶乘**为例演示了递归三要素：
1. **终结点**：`if n == 1: return 1`
2. **递推关系**：`n * factorial(n - 1)`
3. **栈式执行**：逐层深入至终结点，然后逐层返回

```python
factorial(4) -> 4 * factorial(3) -> 3 * factorial(2) -> 2 * factorial(1) -> 1
```

> **注意**：必须有终结点，否则 `RecursionError`。Python 默认递归深度限制约 1000 层。

## 八、常见坑点与对比总结

### 常见坑点

1. **局部变量覆盖全局** --- 函数内赋值不加 `global`，不会影响外部同名变量
2. **可变对象作默认参数** --- `def f(lst=[])` 是经典陷阱，今天仅提及，后续会深入
3. **位置/关键字参数顺序** --- 位置必须在关键字之前，否则 SyntaxError
4. **覆盖内置函数** --- 不要用 `sum`、`list`、`str` 等做变量名
5. **`global` 位置** --- 声明放在函数体最前面（PEP 8）
6. **递归终结点** --- 忘了就 `RecursionError`

### Python vs Java 核心差异

| 维度 | Python | Java |
|------|--------|------|
| 函数地位 | 一等公民（可赋值/传参/独立存在） | 必须属于类（方法） |
| 定义语法 | `def func():` + 缩进 | `public static void func() {}` |
| 多值返回 | `return a, b` -> 元组解包 | 需手动包装类或集合 |
| 默认参数 | 原生支持 | 无（方法重载模拟） |
| `**kwargs` | 原生语法 | 无（Map 或 Builder） |
| 不定长参数 | `*args` 任意类型 | `int... args` 仅同类型 |
