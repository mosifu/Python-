### 2.3 if vs match 选择指南

笔记末尾专门总结了 if 和 match 的选用场景：

| 场景 | 推荐 | 原因 |
|------|:--:|------|
| 范围判断（> <） | if | match 不适合范围 |
| 精确值匹配（==） | match | 代码更清晰 |
| 复杂多条件 | if | if 灵活度更高 |
| 少量分支（2-3） | if | match 杀鸡用牛刀 |
| 多种输入映射 | match | | 多值匹配很优雅 |

## 7. 常见误区与注意事项

### 7.1 笔记中已覆盖的坑点

笔记末尾专门开辟了"常见坑点（Java 背景）"章节，包含 6 条坑点：

| # | 误区 | 说明 |
|:--:|------|------|
| 1 | elif 不是 else if | 写 else if 会报 SyntaxError |
| 2 | 忘了冒号 : | if x > 0 报错，必须 if x > 0: |
| 3 | 缩进不一致 | 同级代码必须对齐，混用空格和 Tab 报错 |
| 4 | match 不需要 break | 每个 case 自动独立，不像 Java switch 会穿透 |
| 5 | match 是 Python 3.10+ | 旧版本不支持的语法 |
| 6 | 链式比较 | 60 <= score < 85 Python 支持，Java 不行 |

### 7.2 未覆盖但值得注意

1. **Python 无 switch 语句**：3.10 以下只能用 if-elif-else，3.10+ 推荐用 match
2. **match 中的变量绑定**：case x: 会匹配任何值并将值绑定到变量 x（与 _ 行为不同）
3. **常量匹配陷阱**：直接在 match 中写变量名会被解释为变量绑定而非常量比较（需要 case 搭配字面量或枚举）
4. **if 条件的真值测试**：Python 中 if non_empty_list: 等价于 if len(list) > 0:，未提及
