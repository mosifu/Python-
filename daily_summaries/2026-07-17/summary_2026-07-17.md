## 7. 常见误区与注意事项

### 7.1 笔记已覆盖的坑点

| 坑点 | 说明 |
|------|------|
| while 忘记更新条件变量 | 导致死循环，检查循环体是否修改了条件 |
| range() 不包含 end | range(1,10) 只到 9，对应 Java 的 i < 10 |
| print() 默认换行 | 同行输出必须加 end="" |
| while...else / for...else | Java 没有此语法，break 跳出后 else 不执行 |
| Python 没有 do-while | 用 while True + break 模拟 |
| 嵌套循环变量命名混淆 | 建议用语义化变量如 row/col |

### 7.2 潜在但未明确指出的坑点

| 坑点 | 说明 |
|------|------|
| continue 在 while 中的位置 | 如果 continue 在条件变量更新之前，可能导致死循环 |
| for 循环中修改被遍历的序列 | 在遍历列表时增删元素会导致意外行为 |
| range() 空序列 | range(10, 0) 步长默认为正，结果为空序列 |
| 嵌套循环中 break 只跳出内层 | 如需跳出外层需使用 flag 变量或 for...else 组合检测 |
