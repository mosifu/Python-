## 核心概念

### 1. 什么是字面量（Literal）

- **是什么**：代码中直接写出来的固定值，就像超市里的价签 — 价签上写的 `3.50` 元就是字面量，你看一眼就知道它是什么值。
- **Java 对比**：
  - 相似点：Java 和 Python 的整数、浮点数、字符串字面量写法几乎一模一样
  - 不同点：Python 无需声明类型，字面量可以直接使用；Python 的 `True`/`False` 首字母**大写**
- **常见字面量一览**：

| 字面量类型 | Python 写法 | Java 写法 | 说明 |
|-----------|------------|----------|------|
| 整数 | `100`, `-5`, `0` | `100`, `-5`, `0` | 完全一样 |
| 浮点数 | `3.14`, `-0.5` | `3.14`, `-0.5` | 完全一样 |
| 字符串 | `"hello"`, `'你好'` | `"hello"` | Python 单双引号均可 |
| 多行字符串 | `"""..."""` | `"""..."""`（文本块） | Python 引入更早，更常用 |
| 布尔值 | `True`, `False` | `true`, `false` | **Python 首字母大写！** |
| 空值 | `None` | `null` | 含义相同，写法不同 |
| 复数 | `1+2j` | 无原生支持 | Python 内置支持 |

---

### 2. 数字字面量（Numeric Literals）

#### 2.1 整数（int）
```python
# 整数 — 与 Java 的 int/long 不同，Python 自动处理大数
age = 25
population = 14_0000_0000  # 可以用下划线分隔，更易读（Java 7+ 也支持）
negative = -10
```

**Java 不同点**：
- Java 有 `int`(4字节)、`long`(8字节) 之分，Python 的 int **无限大**，自动扩容
- Python 不需要 `L` 后缀表示长整型

#### 2.2 浮点数（float）
```python
# 浮点数 — 相当于 Java 的 double（双精度 64 位）
pi = 3.14159
price = -0.5
scientific = 1.5e-3  # 科学计数法 = 0.0015
```

**Java 不同点**：
- Python 只有 `float`（相当于 `double`），没有 `float`/`double` 之分
- Java 的 `float f = 3.14f` 需要 `f` 后缀，Python 不需要

#### 2.3 复数（complex）— Python 独有
```python
# Python 内置复数支持，Java 需要第三方库
c = 1 + 2j
print(c.real)  # 1.0
print(c.imag)  # 2.0
```

---

### 3. 字符串字面量（String Literals）

- **是什么**：文本数据，就像你发的一条微信消息，内容被引号包裹起来。

```python
# Python 字符串 4 种写法
name1 = '单引号字符串'         # 最简洁
name2 = "双引号字符串"         # 和 Java 一样
name3 = '''三单引号
可跨行'''                      # Java 无直接对应
name4 = """三双引号
也可跨行"""                    # ≈ Java 13+ 文本块 Text Block
```

**Java 对比**：

| 特性 | Python | Java |
|------|--------|------|
| 单引号字符串 | ✅ `'hello'` | ❌ 单引号是 `char` |
| 双引号字符串 | ✅ `"hello"` | ✅ `"hello"` |
| 多行字符串 | `'''...'''` 或 `"""..."""` | `"""..."""` (Java 13+) |
| 字面量中引号冲突 | `"He said 'hi'"` 自动解决 | 需要 `\"` 转义 |

---

### 4. 布尔字面量（Boolean Literals）

- **是什么**：只有 `True` 和 `False` 两个值，像开关 — 不是开就是关。

```python
is_student = True   # ✅ 正确：首字母大写
is_teacher = False  # ✅ 正确

# is_student = true  # ❌ 错误！Java 程序员常见坑
```

> ⚠️ **Java 背景常见坑**：`True` / `False`，不是 `true` / `false`！

---

### 5. None — Python 的"空"

- **是什么**：表示"没有值"，像一张空白表格。Java 里叫 `null`，Python 里叫 `None`。

```python
result = None  # 相当于 Java 的 null

# 判断是否为 None，用 is，不用 ==
if result is None:       # ✅ 推荐
    print("无结果")

# if result == None:     # ⚠️ 能运行但不推荐
```

**Java 对比**：

| | Python | Java |
|------|--------|------|
| 写法 | `None` | `null` |
| 类型 | `NoneType` 对象 | 关键字 |
| 判断 | `is None` | `== null` |

---

### 6. 用 `type()` 查看字面量类型

```python
print(type(100))       # <class 'int'>
print(type(3.14))      # <class 'float'>
print(type("hello"))   # <class 'str'>
print(type(True))      # <class 'bool'>
print(type(None))      # <class 'NoneType'>
```

---

## 常见坑点（Java 背景）

1. **`True`/`False` 大写开头** — 写惯了 Java 的 `true`/`false` 极易写错
2. **Python 没有 `char` 类型** — 单个字符 `'a'` 也是 `str`，和 Java 完全不同
3. **`None` 不是 `null`** — 判断用 `is None` 而非 `== null`
4. **整数无限大** — Python 大整数运算不会溢出，和 Java 的 `BigInteger` 一样方便但语法更简洁
5. **单双引号等价** — `'hello'` 和 `"hello"` 完全一样，Java 里单引号是 char

---

## 小结

1. 字面量是代码中直接写出来的固定值，Python 和 Java 的数字/字符串字面量写法高度相似
2. 关键差异记三样：**`True/False` 大写**、**`None` 取代 `null`**、**单双引号等价**
3. 用 `type()` 函数可以随时查看任何值的类型

## 变量与标识符

> 📺 **对应课程**：第 13–14 集

### 1. 什么是变量

- **是什么**：变量就像快递柜的储物格 — 每个格子有编号（变量名），里面可以放东西（值），东西可以随时更换。
- **Java 对比**：
  - 相似点：都是给一块内存区域起名字，用来存数据
  - 不同点：Python **无需声明类型**，直接赋值即可用；同一个变量可以先后指向不同类型的值

```python
# Python：直接赋值，无需声明类型
name = "张三"        # 字符串
age = 20             # 整数
score = 98.5         # 浮点数
is_passed = True     # 布尔值
```

**Java 等价写法**：
```java
String name = "张三";   // 必须声明类型
int age = 20;
double score = 98.5;
boolean isPassed = true;
```

### 2. 标识符命名规则

| 规则 | Python | Java |
|------|--------|------|
| 组成字符 | 字母、数字、下划线 | 字母、数字、下划线、`$` |
| 开头 | **不能数字开头** | 不能数字开头 |
| 大小写敏感 | ✅ `name` ≠ `Name` | ✅ 同 Python |
| 保留字 | `if` `for` `class` 等不能用 | 同 Python |
| 约定风格 | `snake_case`（小写+下划线） | `camelCase` |
| 中文支持 | ⚠️ 允许但不推荐 | 不允许 |

```python
# ✅ 正确命名
user_name = "张三"      # snake_case，Python 推荐风格
_age = 18               # 下划线开头可表示私有
user2 = "李四"          # 可含数字
姓名 = "王五"           # 允许中文，但强烈不推荐

# ❌ 错误命名
# 2user = "错误"       # 不能数字开头
# user-name = "错误"   # 连字符不是下划线
# class = "错误"       # class 是保留字
```

> 💡 **Python 命名约定是 `snake_case`**，不像 Java 用 `camelCase`。写 Python 时跟着习惯走，别写成 `userName`。

### 3. 赋值语句

- **是什么**：`=` 不是"等于号"，而是"绑定"操作 — 把名字贴到东西上。

```python
# 基本赋值
x = 10

# 多重赋值（一次给多个变量赋值）← Java 没有！
a, b, c = 1, 2, 3        # a=1, b=2, c=3

# 交换变量 — Python 一行搞定，不需要临时变量！
a, b = b, a               # 值交换完成

# 链式赋值
x = y = z = 0             # x=0, y=0, z=0
```

**Java 对比 — 交换两个变量**：
```java
// Java：需要临时变量
int temp = a;
a = b;
b = temp;
```

```python
# Python：一行搞定
a, b = b, a
```

## 数据类型

> 📺 **对应课程**：第 15 集

### 1. 动态类型 vs 静态类型

这是 Python 和 Java **最根本**的区别之一：

| | Python（动态类型） | Java（静态类型） |
|------|------|------|
| 变量声明 | `x = 10` 无需声明类型 | `int x = 10;` 必须声明 |
| 类型检查 | **运行时**确定 | **编译时**检查 |
| 类型改变 | 同一变量可换类型 | 声明后不可改变 |
| 语法开销 | 少，代码短 | 多，代码长但 IDE 提示强 |

```python
# Python：同一变量可以先后指向不同类型
data = 100           # 此时是 int
print(type(data))    # <class 'int'>

data = "hello"       # 同一变量，现在变成 str — Java 做不到！
print(type(data))    # <class 'str'>

data = [1, 2, 3]     # 又变成 list 了
print(type(data))    # <class 'list'>
```

> ⚠️ **Java 习惯要注意**：虽然 Python 可以随便换类型，但**实际项目中不建议**这么做，会让代码难以阅读。保持一个变量只用一种类型是良好习惯。

### 2. Python 常见数据类型

```python
# 数字类型
age = 20              # int — 整数
price = 9.99          # float — 浮点数（= Java double）
complex_num = 1+2j    # complex — 复数（Java 没有）

# 文本类型
message = "Hello"     # str — 字符串（无单独的 char 类型）

# 布尔类型
is_ok = True          # bool — 注意大写！

# 空类型
result = None         # NoneType — = Java 的 null
```

**数据类型速查表**：

| 数据类型 | Python | Java 对应 | Python 独有特性 |
|---------|--------|----------|---------------|
| 整数 | `int` | `int` / `long` / `BigInteger` | **无限精度**，自动扩容 |
| 浮点 | `float` | `double` | 等价于 Java `double`（64位） |
| 复数 | `complex` | ❌ 无 | 内置 `1+2j` 语法 |
| 字符串 | `str` | `String` | 无 `char`；单双引号等价 |
| 布尔 | `bool` | `boolean` | 只有 `True`/`False` |
| 空值 | `NoneType` | `null` | `None` 是对象，非关键字 |

### 3. `type()` 与 `isinstance()` — 查类型

```python
# type() — 获取变量的类型
x = 42
print(type(x))            # <class 'int'>
print(type(x) == int)     # True

# isinstance() — 判断是否属于某类型（推荐！）
print(isinstance(x, int))       # True
print(isinstance(x, str))       # False
print(isinstance(x, (int, float)))  # True — 可同时判断多种类型
```

**`type()` vs `isinstance()`**：
- `type()` — 精确匹配，不考虑继承
- `isinstance()` — 考虑继承关系，**更推荐**（和 Java 的 `instanceof` 一样常用）

### 4. 类型转换（Type Casting）

```python
# int() — 转为整数
print(int(3.14))          # 3（截断小数，不是四舍五入）
print(int("100"))         # 100

# float() — 转为浮点数
print(float(10))          # 10.0
print(float("3.14"))      # 3.14

# str() — 转为字符串
print(str(100))           # "100"
print(str(True))          # "True"

# bool() — 转为布尔值
print(bool(1))            # True（非0为True）
print(bool(0))            # False
print(bool(""))           # False（空字符串为False）
print(bool("hello"))      # True（非空字符串为True）
```

**Java 对比**：

| 操作 | Python | Java |
|------|--------|------|
| int → String | `str(100)` | `String.valueOf(100)` 或 `"" + 100` |
| String → int | `int("100")` | `Integer.parseInt("100")` |
| float → int | `int(3.14)` → `3` | `(int) 3.14` → `3` |

## 常见坑点（新增）

6. **变量先赋值后使用** — Python 无默认值，未赋值的变量使用时报 `NameError`
7. **`input()` 返回的永远是 `str`** — 即使用户输入数字，需要 `int()` 转换（Java 程序员容易忘）
8. **Python 是强类型 + 动态类型** — 运行时不自动转换类型，`"3" + 5` 会报错（和 JS 不同，和 Java 一样严格）

---

## 小结

1. Python 变量**无需声明类型**，赋值即创建，推荐用 `snake_case` 命名
2. Python 是**动态类型**语言，类型在运行时确定，同一变量可先后指向不同类型（但不建议滥用）
3. 常用类型转换用 `int()` `float()` `str()` `bool()`；查类型用 `type()` 和 `isinstance()`
