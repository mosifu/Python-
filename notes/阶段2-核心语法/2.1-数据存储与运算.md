## 核心概念

### 1. 什么是字面量（Literal）

- **是什么**：代码中直接写出来的固定值，就像超市里的价签 — 价签上写的 `3.50` 元就是字面量，你看一眼就知道它是什么值。
- **Java 对比**：
  - 相似点：Java 和 Python 的整数、浮点数、字符串字面量写法几乎一模一样
  - 不同点：Python 无需声明类型，字面量可以直接使用；Python 的 `True`/`False` 首字母**大写**
- **常见字面量一览**：

| 字面量类型 | Python 写法 | Java 写法 | 说明 |
|-----------|------------|----------|------|
| 整数 | `100`, `-5`, `0` | `100`, `-5`, `0` | 完全一样 |
| 浮点数 | `3.14`, `-0.5` | `3.14`, `-0.5` | 完全一样 |
| 字符串 | `"hello"`, `'你好'` | `"hello"` | Python 单双引号均可 |
| 多行字符串 | `"""..."""` | `"""..."""`（文本块） | Python 引入更早，更常用 |
| 布尔值 | `True`, `False` | `true`, `false` | **Python 首字母大写！** |
| 空值 | `None` | `null` | 含义相同，写法不同 |
| 复数 | `1+2j` | 无原生支持 | Python 内置支持 |

---

### 2. 数字字面量（Numeric Literals）

#### 2.1 整数（int）
```python
# 整数 — 与 Java 的 int/long 不同，Python 自动处理大数
age = 25
population = 14_0000_0000  # 可以用下划线分隔，更易读（Java 7+ 也支持）
negative = -10
```

**Java 不同点**：
- Java 有 `int`(4字节)、`long`(8字节) 之分，Python 的 int **无限大**，自动扩容
- Python 不需要 `L` 后缀表示长整型

#### 2.2 浮点数（float）
```python
# 浮点数 — 相当于 Java 的 double（双精度 64 位）
pi = 3.14159
price = -0.5
scientific = 1.5e-3  # 科学计数法 = 0.0015
```

**Java 不同点**：
- Python 只有 `float`（相当于 `double`），没有 `float`/`double` 之分
- Java 的 `float f = 3.14f` 需要 `f` 后缀，Python 不需要

#### 2.3 复数（complex）— Python 独有
```python
# Python 内置复数支持，Java 需要第三方库
c = 1 + 2j
print(c.real)  # 1.0
print(c.imag)  # 2.0
```

### 3. 字符串字面量（String Literals）

- **是什么**：文本数据，就像你发的一条微信消息，内容被引号包裹起来。

```python
# Python 字符串 4 种写法
name1 = '单引号字符串'         # 最简洁
name2 = "双引号字符串"         # 和 Java 一样
name3 = '''三单引号
可跨行'''                      # Java 无直接对应
name4 = """三双引号
也可跨行"""                    # ≈ Java 13+ 文本块 Text Block
```

**Java 对比**：

| 特性 | Python | Java |
|------|--------|------|
| 单引号字符串 | ✅ `'hello'` | ❌ 单引号是 `char` |
| 双引号字符串 | ✅ `"hello"` | ✅ `"hello"` |
| 多行字符串 | `'''...'''` 或 `"""..."""` | `"""..."""` (Java 13+) |
| 字面量中引号冲突 | `"He said 'hi'"` 自动解决 | 需要 `\"` 转义 |

---

### 4. 布尔字面量（Boolean Literals）

- **是什么**：只有 `True` 和 `False` 两个值，像开关 — 不是开就是关。

```python
is_student = True   # ✅ 正确：首字母大写
is_teacher = False  # ✅ 正确

# is_student = true  # ❌ 错误！Java 程序员常见坑
```

> ⚠️ **Java 背景常见坑**：`True` / `False`，不是 `true` / `false`！

---

### 5. None — Python 的"空"

- **是什么**：表示"没有值"，像一张空白表格。Java 里叫 `null`，Python 里叫 `None`。

```python
result = None  # 相当于 Java 的 null

# 判断是否为 None，用 is，不用 ==
if result is None:       # ✅ 推荐
    print("无结果")

# if result == None:     # ⚠️ 能运行但不推荐
```

**Java 对比**：

| | Python | Java |
|------|--------|------|
| 写法 | `None` | `null` |
| 类型 | `NoneType` 对象 | 关键字 |
| 判断 | `is None` | `== null` |

### 6. 用 `type()` 查看字面量类型

```python
print(type(100))       # <class 'int'>
print(type(3.14))      # <class 'float'>
print(type("hello"))   # <class 'str'>
print(type(True))      # <class 'bool'>
print(type(None))      # <class 'NoneType'>
```

## 常见坑点（Java 背景）

1. **`True`/`False` 大写开头** — 写惯了 Java 的 `true`/`false` 极易写错
2. **Python 没有 `char` 类型** — 单个字符 `'a'` 也是 `str`，和 Java 完全不同
3. **`None` 不是 `null`** — 判断用 `is None` 而非 `== null`
4. **整数无限大** — Python 大整数运算不会溢出，和 Java 的 `BigInteger` 一样方便但语法更简洁
5. **单双引号等价** — `'hello'` 和 `"hello"` 完全一样，Java 里单引号是 char

---

## 小结

1. 字面量是代码中直接写出来的固定值，Python 和 Java 的数字/字符串字面量写法高度相似
2. 关键差异记三样：**`True/False` 大写**、**`None` 取代 `null`**、**单双引号等价**
3. 用 `type()` 函数可以随时查看任何值的类型

## 变量与标识符

> 📺 **对应课程**：第 13–14 集

### 1. 什么是变量

- **是什么**：变量就像快递柜的储物格 — 每个格子有编号（变量名），里面可以放东西（值），东西可以随时更换。
- **Java 对比**：
  - 相似点：都是给一块内存区域起名字，用来存数据
  - 不同点：Python **无需声明类型**，直接赋值即可用；同一个变量可以先后指向不同类型的值

```python
# Python：直接赋值，无需声明类型
name = "张三"        # 字符串
age = 20             # 整数
score = 98.5         # 浮点数
is_passed = True     # 布尔值
```

**Java 等价写法**：
```java
String name = "张三";   // 必须声明类型
int age = 20;
double score = 98.5;
boolean isPassed = true;
```

### 2. 标识符命名规则

| 规则 | Python | Java |
|------|--------|------|
| 组成字符 | 字母、数字、下划线 | 字母、数字、下划线、`$` |
| 开头 | **不能数字开头** | 不能数字开头 |
| 大小写敏感 | ✅ `name` ≠ `Name` | ✅ 同 Python |
| 保留字 | `if` `for` `class` 等不能用 | 同 Python |
| 约定风格 | `snake_case`（小写+下划线） | `camelCase` |
| 中文支持 | ⚠️ 允许但不推荐 | 不允许 |

```python
# ✅ 正确命名
user_name = "张三"      # snake_case，Python 推荐风格
_age = 18               # 下划线开头可表示私有
user2 = "李四"          # 可含数字
姓名 = "王五"           # 允许中文，但强烈不推荐

# ❌ 错误命名
# 2user = "错误"       # 不能数字开头
# user-name = "错误"   # 连字符不是下划线
# class = "错误"       # class 是保留字
```

> 💡 **Python 命名约定是 `snake_case`**，不像 Java 用 `camelCase`。写 Python 时跟着习惯走，别写成 `userName`。

### 3. 赋值语句

- **是什么**：`=` 不是"等于号"，而是"绑定"操作 — 把名字贴到东西上。

```python
# 基本赋值
x = 10

# 多重赋值（一次给多个变量赋值）← Java 没有！
a, b, c = 1, 2, 3        # a=1, b=2, c=3

# 交换变量 — Python 一行搞定，不需要临时变量！
a, b = b, a               # 值交换完成

# 链式赋值
x = y = z = 0             # x=0, y=0, z=0
```

**Java 对比 — 交换两个变量**：
```java
// Java：需要临时变量
int temp = a;
a = b;
b = temp;
```

```python
# Python：一行搞定
a, b = b, a
```

### 4. `del` 语句 — 删除变量

- **是什么**：`del` 就像把快递柜格子的标签撕掉 — 变量名没了，里面的东西也没人认领了（由垃圾回收机制处理）。
- **Java 对比**：
  - 相似点：都可以"释放"资源
  - 不同点：Java 中变量由 GC 自动回收，没法手动删除一个局部变量；Python 的 `del` 可以**主动解除变量名的绑定**

```python
# del — 删除变量
x = 100
print(x)     # 100

del x        # 删除变量 x
# print(x)  # NameError: name 'x' is not defined — x 已经不存在了！
```

**`del` 不只删变量，还能删容器元素**（学到 2.3 数据容器时会更常用）：

```python
# 删除列表元素
nums = [1, 2, 3, 4]
del nums[0]           # 删除索引 0 的元素
print(nums)           # [2, 3, 4]

# 删除字典键值对
info = {"name": "张三", "age": 20}
del info["age"]
print(info)           # {"name": "张三"}
```

> 💡 **什么时候用 `del`？** 普通变量让 GC 自动处理就行，**不需要刻意 `del`**。`del` 主要用于从列表/字典中删除指定元素。

## 数据类型

> 📺 **对应课程**：第 15 集

### 1. 动态类型 vs 静态类型

这是 Python 和 Java **最根本**的区别之一：

| | Python（动态类型） | Java（静态类型） |
|------|------|------|
| 变量声明 | `x = 10` 无需声明类型 | `int x = 10;` 必须声明 |
| 类型检查 | **运行时**确定 | **编译时**检查 |
| 类型改变 | 同一变量可换类型 | 声明后不可改变 |
| 语法开销 | 少，代码短 | 多，代码长但 IDE 提示强 |

```python
# Python：同一变量可以先后指向不同类型
data = 100           # 此时是 int
print(type(data))    # <class 'int'>

data = "hello"       # 同一变量，现在变成 str — Java 做不到！
print(type(data))    # <class 'str'>

data = [1, 2, 3]     # 又变成 list 了
print(type(data))    # <class 'list'>
```

> ⚠️ **Java 习惯要注意**：虽然 Python 可以随便换类型，但**实际项目中不建议**这么做，会让代码难以阅读。保持一个变量只用一种类型是良好习惯。

### 2. Python 常见数据类型

```python
# 数字类型
age = 20              # int — 整数
price = 9.99          # float — 浮点数（= Java double）
complex_num = 1+2j    # complex — 复数（Java 没有）

# 文本类型
message = "Hello"     # str — 字符串（无单独的 char 类型）

# 布尔类型
is_ok = True          # bool — 注意大写！

# 空类型
result = None         # NoneType — = Java 的 null
```

**数据类型速查表**：

| 数据类型 | Python | Java 对应 | Python 独有特性 |
|---------|--------|----------|---------------|
| 整数 | `int` | `int` / `long` / `BigInteger` | **无限精度**，自动扩容 |
| 浮点 | `float` | `double` | 等价于 Java `double`（64位） |
| 复数 | `complex` | ❌ 无 | 内置 `1+2j` 语法 |
| 字符串 | `str` | `String` | 无 `char`；单双引号等价 |
| 布尔 | `bool` | `boolean` | 只有 `True`/`False` |
| 空值 | `NoneType` | `null` | `None` 是对象，非关键字 |

### 3. `type()` 与 `isinstance()` — 查类型

```python
# type() — 获取变量的类型
x = 42
print(type(x))            # <class 'int'>
print(type(x) == int)     # True

# isinstance() — 判断是否属于某类型（推荐！）
print(isinstance(x, int))       # True
print(isinstance(x, str))       # False
print(isinstance(x, (int, float)))  # True — 可同时判断多种类型
```

**`type()` vs `isinstance()`**：
- `type()` — 精确匹配，不考虑继承
- `isinstance()` — 考虑继承关系，**更推荐**（和 Java 的 `instanceof` 一样常用）

### 4. 类型转换（Type Casting）

```python
# int() — 转为整数
print(int(3.14))          # 3（截断小数，不是四舍五入）
print(int("100"))         # 100

# float() — 转为浮点数
print(float(10))          # 10.0
print(float("3.14"))      # 3.14

# str() — 转为字符串
print(str(100))           # "100"
print(str(True))          # "True"

# bool() — 转为布尔值
print(bool(1))            # True（非0为True）
print(bool(0))            # False
print(bool(""))           # False（空字符串为False）
print(bool("hello"))      # True（非空字符串为True）
```

**Java 对比**：

| 操作 | Python | Java |
|------|--------|------|
| int → String | `str(100)` | `String.valueOf(100)` 或 `"" + 100` |
| String → int | `int("100")` | `Integer.parseInt("100")` |
| float → int | `int(3.14)` → `3` | `(int) 3.14` → `3` |

## 字符串操作

> 📺 **对应课程**：第 16–17 集

### 1. 字符串拼接

- **是什么**：把多段文字连成一句话，就像拼积木 — 把零散的词拼成完整的句子。
- **Java 对比**：
  - 相似点：`+` 拼接和 Java 的 `String +` 完全一样
  - 不同点：Python 支持**隐式拼接**（相邻字符串自动合并）；`+` 只能拼字符串，不能自动把数字转字符串

```python
# 方式1：+ 拼接（最常用）
name = "张三"
age = 20
# print("我是" + name + "，今年" + age)  # ❌ 报错！age 是 int，不能直接+
print("我是" + name + "，今年" + str(age) + "岁")  # ✅ 用 str() 转换

# 方式2：隐式拼接 — 相邻字符串字面量自动合并（Java 没有！）
msg = "气温二十一度" '刚好又是花季' """适合出去玩"""
print(msg)  # 气温二十一度刚好又是花季适合出去玩

# 方式3：join() — 用分隔符连接列表（学到 2.3 会细讲）
words = ["Hello", "Python", "World"]
print(" ".join(words))  # Hello Python World
```

> ⚠️ **Java 习惯坑**：Python 不会像 Java 那样自动 `toString()`，`+` 两边必须都是 `str`，否则报 `TypeError`。

### 2. 字符串格式化

| 方式 | 语法 | 推荐度 | Java 类比 |
|------|------|--------|----------|
| `%s` 占位 | `"我叫%s，今年%d岁" % (name, age)` | ⭐⭐ | `String.format()` |
| `f-string` | `f"我叫{name}，今年{age}岁"` | ⭐⭐⭐ | Java 无直接对应，最接近 `STR."..."` (Java 21+) |
| `.format()` | `"我叫{}，今年{}".format(name, age)` | ⭐ | `MessageFormat.format()` |

```python
name = "mo_sifu"
age = 23
job = "AI"
hobby = "run"

# 方式1：%s 占位（老式，但课程中用得多）
print("我是%s，今年%s岁，专业%s，爱好%s" % (name, age, job, hobby))

# 方式2：f-string（推荐！）— 变量直接写花括号里
print(f"我是{name}，今年{age}岁，专业{job}，爱好{hobby}")

# f-string 里面还能写表达式
print(f"明年我就{age + 1}岁了")        # 28
print(f"名字大写：{name.upper()}")      # MO_SIFU

# f-string 格式化数字
pi = 3.14159
print(f"π ≈ {pi:.2f}")                # 保留 2 位小数 → 3.14
print(f"百分比：{0.856:.1%}")           # 85.6%
```

> 💡 **推荐 f-string**：简洁、直观、性能好。`%s` 了解即可（课程里会用）。

### 3. 转义字符

```python
# 常用转义字符
print("It's a good day")       # 引号自动识别，无需转义
print('It\'s a good day')      # 同引号时用 \' 转义

print("你好世界，\n欢迎来到Python的世界")  # \n 换行
print("姓名\t年龄\t城市")                  # \t 制表符（Tab）
```

**Java 对比**：`\n` `\t` `\\` 等和 Java 完全一样，不用重新学。

## 输入输出

> 📺 **对应课程**：第 18–19 集

### 1. `input()` — 获取用户输入

- **是什么**：程序停下来等用户敲键盘，就像一个对话框问"请输入："。
- **Java 对比**：
  - 相似点：都是从控制台读取用户输入
  - 不同点：Python 只需一行 `input()`；Java 需要 `Scanner` + `System.in`

```python
# Python — 一行搞定
name = input("请输入您的姓名：")
print(f"你好，{name}！")

# Java 等价写法：
# Scanner scanner = new Scanner(System.in);
# System.out.print("请输入您的姓名：");
# String name = scanner.nextLine();
```

> ⚠️ **最重要的坑**：`input()` 返回的永远是 `str`，做数学运算前必须用 `int()` 或 `float()` 转换！

```python
# ❌ 错误：input 返回字符串，+ 变成拼接！
a = input("输入 a：")   # 用户输入 10
b = input("输入 b：")   # 用户输入 20
print(a + b)           # "1020" ← 字符串拼接，不是 30！

# ✅ 正确：先转换类型
a = int(input("输入 a："))
b = int(input("输入 b："))
print(a + b)           # 30 ← 数值加法
```

### 2. `print()` — 输出到控制台

```python
# 基本用法
print("Hello World")

# f-string 配合 print（最佳搭档）
name = "小红"
score = 95
print(f"{name} 的成绩是 {score} 分")

# print 的 end 参数 — 默认换行，可改成不换行
print("第一行", end=" → ")
print("第二行")   # 输出：第一行 → 第二行
```

### 3. 实战案例：模拟 ATM 取款

```python
# 综合运用 input + print + 类型转换
balance = 10000
password = input("请输入密码：")
if password == "123456":
    amount = int(input("请输入取款金额："))
    balance = balance - amount
    print(f"取款成功！取款{amount}元，余额{balance}元")
else:
    print("密码错误！")
```

## 运算符

> 📺 **对应课程**：第 20–21 集

### 1. 算术运算符

| 运算符 | 含义 | 示例 | 结果 | Java 相同？ |
|--------|------|------|------|:----------:|
| `+` | 加 | `10 + 4` | `14` | ✅ |
| `-` | 减 | `10 - 4` | `6` | ✅ |
| `*` | 乘 | `10 * 4` | `40` | ✅ |
| `/` | 除 | `10 / 4` | `2.5` | ⚠️ 不同！Java 整数除得整数 `2` |
| `%` | 取余 | `10 % 4` | `2` | ✅ |
| `//` | 整除 | `10 // 4` | `2` | ≈ Java `/` 整数除法 |
| `**` | 指数 | `10 ** 4` | `10000` | ❌ Java 需要 `Math.pow()` |

> ⚠️ **Java 最大不同**：`/` 在 Python 中总是返回浮点数 `2.5`，不像 Java `int/int` 返回 `int`。Python 用 `//` 做整除。

```python
print(f"10 + 4 = {10 + 4}")    # 14
print(f"10 - 4 = {10 - 4}")    # 6
print(f"10 * 4 = {10 * 4}")    # 40
print(f"10 / 4 = {10 / 4}")    # 2.5（Java 中是 2！）
print(f"10 % 4 = {10 % 4}")    # 2
print(f"10 // 4 = {10 // 4}")  # 2（整除，等同 Java int/int）
print(f"10 ** 4 = {10 ** 4}")  # 10000
```

### 2. 运算符优先级

```
从高到低：
    ① 括号 ()
    ② **      （指数）
    ③ * / % // （乘除）
    ④ + -     （加减）
```

```python
print(0.1 + 10 / 4 ** 2)   # 0.1 + 10 / 16 = 0.1 + 0.625 = 0.725
print((0.1 + 10) / 4)      # 10.1 / 4 = 2.525（括号改变优先级）
```

### 3. 浮点数精度问题

```python
print(0.5 - 0.4)  # 0.09999999999999998 ← 不是 0.1！
```

> 💡 这不是 Python 的 bug，所有语言（包括 Java）都有二进制浮点精度问题。`0.1` 无法用有限的二进制精确表示，就像 `1/3` 无法用十进制精确表示一样。

### 4. 赋值运算符

| 运算符 | 等价于 | 说明 |
|--------|--------|------|
| `=` | `x = 10` | 基本赋值 |
| `+=` | `x = x + 10` | 加后赋值 |
| `-=` | `x = x - 10` | 减后赋值 |
| `*=` | `x = x * 10` | 乘后赋值 |
| `/=` | `x = x / 10` | 除后赋值 |
| `//=` | `x = x // 10` | 整除后赋值 |
| `%=` | `x = x % 10` | 取余后赋值 |
| `**=` | `x = x ** 10` | 指数后赋值 |

```python
num = 85
num += 10   # 95
num -= 10   # 85
num *= 10   # 850
num /= 10   # 85.0
num //= 10  # 8.0
num %= 3    # 2.0
num **= 3   # 8.0
```

**Java 对比**：`+=` `-=` `*=` `/=` `%=` 和 Java 完全一样，多了 `//=` 和 `**=`。

### 5. 比较运算符

| 运算符 | 含义 | Java 相同？ |
|--------|------|:----------:|
| `==` | 等于 | ✅ |
| `!=` | 不等于 | ✅ |
| `>` | 大于 | ✅ |
| `<` | 小于 | ✅ |
| `>=` | 大于等于 | ✅ |
| `<=` | 小于等于 | ✅ |

**和 Java 完全一样**，无记忆负担。

```python
print(100 == 100)   # True
print(100 != 100)   # False
print(100 >= 100)   # True
print(100 < 100)    # False
```

### 6. 逻辑运算符

| Python | Java | 含义 |
|--------|------|------|
| `and` | `&&` | 与 |
| `or` | `||` | 或 |
| `not` | `!` | 非 |

> ⚠️ **Java 最大坑**：Python 用英文单词 `and`/`or`/`not`，不是 `&&`/`||`/`!`！

```python
# 判断数字是否在 10–20 之间
num = int(input("请输入一个整数："))
print(f"{num} 在 10–20 之间吗？ {num >= 10 and num <= 20}")

# 判断数字是否不在 10–20 之间
print(f"{num} 不在 10–20 之间吗？ {num < 10 or num > 20}")

# not 取反
is_student = True
print(not is_student)  # False
```

## 常见坑点（下篇：字符串+输入+运算符）

9. **`+` 不自动转字符串** — `"年龄" + 18` 报 TypeError，和 Java 自动 `toString()` 不同
10. **`/` 总是返回 float** — `10 / 4` = `2.5`，取整除用 `//`
11. **`and`/`or`/`not` 不是 `&&`/`||`/`!`** — Java 程序员极高频率写错
12. **浮点精度问题** — `0.1 + 0.2 != 0.3`，金融计算用 `Decimal`
13. **`input()` 返回 `str`** — 做数学前必须 `int()` 或 `float()` 转换

---

## 小结

1. 字符串拼接用 `+`（需 `str()` 转换）或 `f-string`（推荐）；格式化优先用 `f"{变量}"`
2. `input()` 返回永远是 `str`，计算前用 `int()`/`float()` 转换；`print()` + `f-string` 是最佳输出组合
3. 运算符：`/` 总返回 float、`//` 整除、`**` 指数、`and`/`or`/`not` 替代 `&&`/`||`/`!`
